# 알림 화면 PRD 스펙 검증 리포트

> **검증일**: 2026-02-01  
> **대상**: `notification-center`, `notification-settings` 화면  
> **결과**: P1 기준 **85%** 적합도 ✅

---

## 1. 알림 센터 (Notification Center)

### PRD 스펙 vs 프로토타입

| 항목 | PRD 스펙 | 상태 |
|------|----------|------|
| 헤더 (←, 타이틀, 전체 읽음) | 뒤로가기, "알림", 전체 읽음 버튼 | ✅ |
| 알림 카드 요소 | 아이콘, 제목, 내용, 시간, 읽음 표시 | ✅ |
| 삭제 기능 | 스와이프 → 삭제 버튼 | ⚠️ 🗑️ 버튼으로 대체 |
| 알림 클릭 → 이동 | 해당 포트폴리오로 이동 + 읽음 처리 | ⚠️ Alert으로 시뮬레이션 |
| Pull to Refresh | 최신 알림 로드 | ❎ 프로토타입 미구현 |
| 상태 (로딩/빈 화면) | 스켈레톤, 빈 상태 메시지 | ✅ |

---

## 2. 알림 설정 (Notification Settings)

### PRD 스펙 vs 프로토타입

| 항목 | PRD 스펙 | 상태 |
|------|----------|------|
| 헤더 (←, 타이틀) | 뒤로가기, "알림 설정" | ✅ |
| 전체 알림 토글 | ON/OFF, OFF 시 비활성화 | ✅ |
| 알림 주기 | 매일/매주/2주/한 달 | ✅ |
| 알림 시간 피커 | 09:00~21:00 선택 | ⚠️ prompt()로 대체 |
| 포트폴리오별 설정 | 이름, 임계값(1~20%), 토글 | ✅ |
| 임계값 바텀시트 | 슬라이더로 변경 | ✅ |
| 로딩/저장 상태 | 스켈레톤, 토스트 | ❎ 프로토타입 미구현 |

---

## 3. 프로토타입에서 의도적으로 생략한 항목

| 항목 | 이유 |
|------|------|
| Pull to Refresh | 실제 API 연동 시 구현 필요, 프로토타입에서 불필요 |
| 로딩/저장 상태 | Mock 데이터는 즉시 반응하므로 불필요 |
| 스와이프 삭제 | 버튼 삭제로 동일 기능 제공 |

---

## 4. 데이터 모델 차이 (API 연동 시 조정 필요)

| DB 스펙 | 프로토타입 Mock | 비고 |
|---------|-----------------|------|
| `notification_type_id` (FK) | `type` (string) | 타입 정규화 필요 |
| `is_read` (boolean) | `unread` (boolean) | 필드명 반전 |
| `created_at` (timestamp) | `time` (상대 문자열) | 형식 변환 필요 |
| `related_entity_id` | 미구현 | 알림 클릭 시 이동에 필요 |

## 5. 결론

프로토타입은 PRD UI 스펙의 핵심 기능을 충실히 구현함:

- ✅ 알림 목록 표시 및 읽음/삭제 관리
- ✅ 알림 설정 전체/개별 토글, 주기, 시간, 임계값
- ✅ 상태 관리 (로딩, 빈 상태)

실제 개발 시 추가 구현 필요:

- 알림 클릭 → 포트폴리오 이동 연결 (`related_entity_id`)
