# 3. UI/UX 요구사항

### 3-1. 화면 구성 및 사용자 흐름

- **화면 설계서/와이어프레임**:
    - 도구: `Figma 사용 예정`
    - 작성 상태: `Phase 0에서 작성 예정 (2026년 1월 중순)`
    - 포함 내용: `주요 화면 레이아웃, 버튼 배치, 사용자 플로우, 상호작용 디자인`
- **주요 화면 목록** (Phase 1):
    1. **로그인/회원가입 화면**
        - Google 소셜 로그인 버튼
        - 앱 로고 및 간단한 소개 문구
    2. **프로필 입력 화면** (최초 로그인 시)
        - 닉네임 또는 표시명 입력
        - 선택 사항: 프로필 사진 업로드
    3. **포트폴리오 목록 화면** (홈 화면)
        - 포트폴리오 카드 형태로 표시 (이름, 총 평가금액, 수익률)
        - 포트폴리오 추가 버튼 (+ 플로팅 버튼)
        - 상단: 프로필/설정 아이콘
    4. **포트폴리오 상세 화면**
        - 헤더: 포트폴리오 이름, 편집 버튼
        - 요약 정보: 총 평가금액, 총 매입금액, 수익률, 수익금
        - 종목 목록 테이블 (종목명, 보유수량, 현재가, 평가금액, 현재비율, 목표비율)
        - 하단: 종목 추가 버튼, 리밸런싱 분석 버튼
    5. **종목 검색/추가 화면**
        - 검색창 (종목명 또는 종목코드)
        - 검색 결과 리스트 (종목명, 종목코드, 현재가)
        - 선택 시: 보유수량 입력 모달, 목표비율 설정
    6. **리밸런싱 분석 화면**
        - 현재 비율 vs 목표 비율 비교 차트/그래프
        - 리밸런싱 제안 카드 (종목별 매수/매도 수량 및 금액)
        - 임계값 설정 버튼
    7. **설정 화면**
        - 프로필 정보 수정
        - 알림 설정 (ON/OFF, 주기, 시간)
        - 로그아웃
        - 버전 정보, 이용약관, 개인정보처리방침
- **사용자 플로우**:
    1. 앱 실행 → 로그인 화면
    2. Google 로그인 → 프로필 입력 (최초 1회)
    3. 포트폴리오 목록 화면 (홈)
    4. 포트폴리오 선택 → 상세 화면
    5. 종목 추가 → 검색 → 수량/목표비율 입력 → 저장
    6. 리밸런싱 분석 확인 → 조정 수행

### 3-2. 반응형 디자인

- **플랫폼 지원 범위** (Phase 1):
    - 주 타겟: `모바일 (iOS/Android)`
    - 화면 크기: `375px~428px (iPhone SE ~ iPhone Pro Max)`
    - 태블릿 지원: `Phase 2에서 검토`
    - 데스크톱/웹: `Phase 3에서 하이브리드 앱으로 확장 예정`
- **화면 방향**:
    - 기본: `세로 모드 (Portrait)`
    - 가로 모드: `미지원 (Phase 1)`

### 3-3. 디자인 시스템

- **컴포넌트 라이브러리**:
    - 기본 컴포넌트: `React Native 기본 컴포넌트 (Button, Text, View, ScrollView 등)`
    - UI 라이브러리: `React Native Paper 또는 NativeBase 검토 (Phase 1에서 결정)`
- **컴포넌트 명세**:
    - [ ]  Button (기본, Primary, Secondary, Outline)
    - [ ]  Input Field (텍스트, 숫자, 검색)
    - [ ]  Card (포트폴리오, 종목, 리밸런싱 제안)
    - [ ]  Table/List (종목 목록, 거래 내역)
    - [ ]  Modal/Bottom Sheet (입력, 확인, 선택)
    - [ ]  Chart/Graph (비율 차트, 수익률 그래프 - Phase 2)
    - [ ]  Toast/Snackbar (알림 메시지)
    - [ ]  Loading Indicator (스피너, 스켈레톤)
- **컴러 팔레트**:
    - Primary Color: `팀 논의 필요 (예: #1E88E5 - 파란색)`
    - Secondary Color: `팀 논의 필요 (예: #FFA726 - 주황색)`
    - Success: `#66BB6A (초록색 - 수익, 매수)`
    - Error/Danger: `#EF5350 (빨간색 - 손실, 매도)`
    - Warning: `#FFA726 (주황색 - 경고)`
    - Background: `#FFFFFF (흰색 기본), #F5F5F5 (회색 배경)`
    - Text: `#212121 (진한 회색), #757575 (회색), #BDBDBD (연한 회색)`
- **타이포그래피**:
    - 기본 폰트: `Noto Sans KR (Google Fonts) 또는 시스템 기본 폰트`
    - 폰트 크기:
        - Heading 1: `24px, Bold`
        - Heading 2: `20px, Bold`
        - Body: `16px, Regular`
        - Caption: `14px, Regular`
        - Small: `12px, Regular`
- **아이콘**:
    - 라이브러리: `React Native Vector Icons (Material Icons 또는 Ionicons)`
    - 사용 예: 홈, 설정, 추가, 삭제, 검색, 새로고침, 알림 등

### 3-4. 상호작용 디자인

- **터치/제스처**:
    - 탭: `화면 전환, 버튼 클릭`
    - 스와이프: `종목 삭제 (왼쪽으로 스와이프)`
    - 드래그: `Phase 2에서 포트폴리오/종목 순서 변경 기능 추가 검토`
    - Pull to Refresh: `포트폴리오 목록/상세 화면에서 아래로 당겨 실시간 시세 업데이트`
- **피드백**:
    - 버튼 클릭: `짧은 진동(Haptic) + 버튼 색상 변화`
    - 로딩: `스피너 표시 + '불러오는 중...' 텍스트`
    - 성공: `초록색 Toast 메시지 + 체크 아이콘`
    - 실패/에러: `빨간색 Toast 메시지 + 경고 아이콘`
- **애니메이션**:
    - 화면 전환: `슬라이드 애니메이션 (200-300ms)`
    - 모달 표시: `Fade In + Scale Up (300ms)`
    - 리스트 아이템: `Fade In 순차 등장 (100ms 간격)`
    - 버튼 호버: `Scale 약간 확대 (1.05x)`

### 3-5. 접근성 (Accessibility)

- **기본 요구사항**:
    - [ ]  터치 타겟 크기: `최소 44x44pt (iOS), 48x48dp (Android)`
    - [ ]  컴러 대비: `WCAG AA 수준 (4.5:1 이상)`
    - [ ]  폰트 크기 조절: `시스템 폰트 크기 설정 따름`
    - [ ]  Screen Reader 지원: `주요 버튼과 텍스트에 accessibility label 추가`
- **WCAG 준수**:
    - Phase 1: `기본 요구사항만 충족 (AA 수준 목표)`
    - Phase 2: `전체 WCAG 2.1 AA 수준 준수 검토`
- **다국어 지원**:
    - Phase 1: `한국어만 지원`
    - Phase 2: `영어 지원 검토 (i18n 라이브러리 사용)`

### 3-6. 특수 상황 UI

- **빈 상태 (Empty State)**:
    - 포트폴리오 없음: `일러스트 + '첫 포트폴리오를 만들어보세요' + 추가 버튼`
    - 종목 없음: `'종목을 추가해주세요' 메시지`
    - 검색 결과 없음: `'검색 결과가 없습니다' + 다른 키워드 제안`
- **오류 상태 (Error State)**:
    - 네트워크 오류: `오프라인 아이콘 + '인터넷 연결을 확인해주세요' + 재시도 버튼`
    - 서버 오류: `오류 아이콘 + '일시적인 오류가 발생했습니다' + 재시도 버튼`
    - 데이터 로드 실패: `재시도 버튼 표시`
- **로딩 상태 (Loading State)**:
    - 전체 화면 로딩: `중앙 스피너 + 로고`
    - 부분 로딩: `해당 영역에 스켈레톤 UI 또는 작은 스피너`
    - Pull to Refresh: `상단에 로딩 인디케이터`
- **성공 상태 (Success State)**:
    - 저장 성공: `초록색 Toast '저장되었습니다'`
    - 삭제 성공: `Toast '삭제되었습니다' + Undo 버튼 (Phase 2)`
