# [UPDATE] Auth UI Modernization

## 1. 개요

`_inbox/pages/`의 새로운 디자인 시스템에 맞추어 `auth` 도메인의 UI 스펙을 현대화합니다. 로그인 화면의 디자인을 개선하고, 설정(Settings) 메뉴에서 진입하는 프로필 편집 화면을 신규 정의합니다.

## 2. 변경 사항 요약

- **[MODIFY] specs/ui/auth/login-screen.md**: 기능 하이라이트 섹션 추가 및 레이아웃 개선
- **[MODIFY] specs/ui/auth/profile-input.md**: (Onboarding) 프로필 편집 화면과 톤앤매너 통일
- **[NEW] specs/ui/auth/profile-edit.md**: (Settings) 프로필 정보 수정 화면 신규 정의

## 4. AI 분석 (Chain of Thought)

1.  **로그인 화면**: 단순한 "Google Login" 버튼뿐만 아니라, 앱의 가치를 전달하는 "기능 하이라이트" 섹션이 추가된 점이 긍정적입니다. 이는 비회원에게 가입 유도를 강화합니다.
2.  **프로필 편집**: 기존에는 온보딩용 `profile-input`만 있었으나, 앱 사용 도중 닉네임을 변경하려는 니즈(`profile-edit`)를 반영하여 별도 파일로 분리했습니다. 두 화면은 UI 컴포넌트(아바타, 인풋 등)를 공유해야 하므로 일관성을 유지하도록 작성했습니다.
3.  **아바타 정책**: Google 프로필 사진을 우선하되, 로드 실패나 미설정 시 "닉네임 첫 글자"를 보여주는 Fallback UI가 `_inbox` 디자인에 포함됨을 확인했습니다.

## 5. 🔍 확인 필요 사항

### 1. 프로필 이미지 변경 기능 (P1)
디자인 시안에는 카메라 아이콘이나 "사진 변경하기" 버튼이 보이지 않습니다. P1에서는 프로필 이미지 변경(업로드)을 지원하지 않고 Google 계정 사진만 사용하는 것이 맞습니까?

- [ ] **Google 사진만 사용 (P1)**: 별도 업로드 기능 없음 (현재 제안)
- [ ] **커스텀 업로드 지원**: API 및 UI 추가 필요

### 2. 닉네임 중복 체크 시점
닉네임 입력 시 중복 여부를 언제 확인할까요?

- [ ] **저장(시작하기) 버튼 클릭 시**: API 호출 후 실패하면 에러 표시 (현재 제안)
- [ ] **입력 중 실시간(Debounce)**: 별도 "중복 확인" API 필요
