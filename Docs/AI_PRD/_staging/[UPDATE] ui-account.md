# [UPDATE] Account UI Modernization

## 1. 개요

`_inbox/pages/`의 새로운 디자인 시스템에 맞추어 기존 Account 도메인의 UI 스펙(`list.md`, `create.md`)을 현대화하고, 설정 메뉴(`07_settings.md`)와의 연결성을 확보합니다.

## 2. 변경 사항 요약

- **[MODIFY] specs/ui/account/list.md**: 새로운 카드형 레이아웃 및 헤더 스타일 적용
- **[MODIFY] specs/ui/account/create.md**: 새로운 입력 폼 스타일 적용
- **[CONTEXT]**: 설정 페이지(`07_settings.md`)에서 "계좌 관리" 진입점이 필요함

## 3. 상세 스펙 제안

### A. [MODIFY] specs/ui/account/list.md

```markdown

## 개요

증권사 연동 계좌 목록 표시 및 관리

| 조건 | 동작 |
```

┌─────────────────────────────────────┐

├─────────────────────────────────────┤
│                                     │

│ │   ₩5,000,000                  │ │
│ └─────────────────────────────────┘ │
│                                     │
│ ┌─────────────────────────────────┐ │

```
### 헤더
- 타이틀: "계좌 관리"
- 뒤로가기 (←): 설정 화면으로 이동

- **상단**: 증권사 로고/이름
- **중단**: 마스킹된 계좌번호 + 보유 종목 수
- **하단**: 예수금(현금) 잔액

## 상호작용
- **카드 탭**: 계좌 상세(또는 편집) 화면 이동
- **추가 버튼**: `create.md` 화면으로 이동
- **롱프레스**: 삭제/수정 옵션 표시
```

## 4. AI 분석 (Chain of Thought)

1.  **디자인 일관성**:
    - 기존 `list.md`, `create.md`는 예전 스타일(`┌─┐`)이었습니다.
    - 새로운 스타일(`┌───┐` 37자 폭, 헤더 `←` 스타일)로 통일하여 앱의 완성도를 높였습니다.

2.  **Navigation Flow 연결**:
    - 현재 `_inbox/pages/07_settings.md`에는 "계좌 관리" 메뉴가 명시되어 있지 않습니다.
    - 그러나 Account 도메인이 존재하려면 진입점이 필수적이므로, 설정 메뉴 하위에 위치하는 것을 전제로 문서를 작성했습니다.
    - 사용자는 `Account`를 `Portfolio`의 하위 개념으로 볼 수도 있지만, `specs/db/account`가 별도로 존재하므로 "자산 연동 관리" 측면에서 설정에 위치하는 것이 적절합니다.

3.  **데이터 표현**:
    - `03_home.md`에서 현금(₩2,000,000)을 보여주려면 `accounts` 테이블의 데이터가 필수적입니다.
    - 따라서 Account UI는 단순 관리가 아니라, 홈 화면 데이터의 원천을 관리하는 중요한 기능입니다.
