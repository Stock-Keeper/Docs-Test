# [NEW] ui-ads

## 1. 개요

앱 전반에 노출되는 4가지 광고 컴포넌트의 UI 및 동작 명세입니다.

## 2. 컴포넌트 상세

### 2.1. Splash Interstitial (전면 광고)

- **위치**: 앱 콜드 스타트(Cold Start) 시, 메인 화면 진입 전
- **동작 흐름**:
    1. 앱 실행 및 `GET /api/v1/ads/config` 호출 (API 응답 대기, 타임아웃 3초)
    2. `SPLASH` 설정 활성화 확인 시 광고 로드 시작
    3. 로드 완료 → 전면 광고 표시
    4. 광고 닫기 또는 스킵 → 메인 화면 진입
    5. 로드 실패/타임아웃/비활성화 → 즉시 메인 화면 진입
- **빈도 제어**: 로컬 저장소에 마지막 노출 시간/횟수 기록하여 `frequency_cap` 준수

### 2.2. Bottom Banner (하단 배너)

- **위치**: 메인 홈, 관심 종목, 자산 현황 탭 하단
- **형태**: 화면 너비에 맞춘 320x50 또는 Smart Banner
- **동작**: 페이지 이동 간에도 끊김 없이 유지 (Single Activity 구조 권장)

### 2.3. Feed Native Ad (네이티브 인피드)

- **위치**: 커뮤니티 게시판 리스트 사이
- **형태**: 주변 게시물 카드와 유사한 레이아웃 (썸네일 우측/좌측 배치 등 통일)
- **동작**:
    - 리스트 어댑터에서 `config.interval` (예: 5) 번째 아이템마다 광고 뷰 홀더 삽입
    - **[필수]** '광고' 또는 'Ad' 라벨 명시 (구글 정책 준수)

### 2.4. Notification Sticky Banner (상단 고정)

- **위치**: 알림 센터 페이지 상단 헤더 바로 아래
- **형태**: 가로로 긴 배너 (높이 고정)
- **동작**: 알림 리스트 스크롤 시에도 상단 레이아웃에 고정되어 노출됨

## 🔍 확인 필요 사항

### 1. 스플래시 타임아웃 정책

- [x] **3초 (Soft)**: 3초 내 로드 안 되면 포기하고 메인 진입 (사용자 경험 중시)
- [ ] **5초 (Hard)**: 수익 위해 조금 더 기다림

### 2. 네이티브 광고 UI 구현

- [ ] **Google Template**: 구글 제공 기본 템플릿 사용 (구현 10분, 디자인 이질감 있음)
- [x] **Custom Layout**: 게시물 UI와 100% 동일하게 직접 구현 (구현 4시간, 클릭률 높음)
