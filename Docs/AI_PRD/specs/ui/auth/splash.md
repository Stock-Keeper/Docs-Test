# 스플래시 화면 (Splash)

## 개요

앱 실행 시 최초 진입 화면, 토큰 유무로 다음 화면 분기

## 레이아웃

```text
┌─────────────────────────────────┐
│                                 │
│                                 │
│                                 │
│                                 │
│         [앱 로고 이미지(📊)]        │
│                                 │
│           Stock Keeper          │
│                                 │
│                                 │
│                                 │
│                                 │
│            ○ ○ ○               │
│       (로딩 인디케이터)         │
│                                 │
└─────────────────────────────────┘
```

## 컴포넌트

### 1. 앱 로고

| 속성 | 값 |
|---|---|
| 크기 | 120x120 px |
| 아이콘 | 📊 (72px) |
| 위치 | 화면 뷰포트 중앙 정렬 보정 (`justify-content: center`) |
| 애니메이션 | 페이드인 효과 (`logoFadeIn 0.8s ease-out`) |

### 2. 앱 이름

| 속성 | 값 |
|---|---|
| 텍스트 | "Stock Keeper" |
| 폰트 | 24px, Bold(700), Text Primary |
| 위치 | 로고 하단, 마진 80px |
| 애니메이션 | 상향 페이드인 (`titleFadeIn 0.8s ease-out 0.2s both`) |

### 3. 로딩 인디케이터

| 속성 | 값 |
|---|---|
| 형태 | 점 3개 (`10x10px`, Primary 컬러) |
| 위치 | 화면 하단 (`bottom: 120px`), 절대 위치 정렬 |
| 애니메이션 | 맥박 효과 (`dotPulse 1.4s infinite ease-in-out both`, 각 점별 지연시간 적용) |

## 동작 로직

```text
1. 앱 실행
2. 로컬 스토리지에서 Access Token 확인
3. 토큰 존재 → 토큰 유효성 검증 (API 호출 or JWT 만료 확인)
   - 유효 → 홈 화면으로 이동
   - 만료 → Refresh Token으로 갱신 시도
     - 성공 → 홈 화면
     - 실패 → 로그인 화면
4. 토큰 없음 → 로그인 화면으로 이동
5. 최소 표시 시간: 1초 (UX를 위한 지연)
```

## 상태

| 상태 | 동작 |
|---|---|
| 토큰 확인 중 | 로딩 인디케이터 표시 |
| 자동 로그인 성공 | 홈 화면 이동 |
| 토큰 갱신 중 | 로딩 유지 |
| 인증 실패 | 로그인 화면 이동 |
